What happens when you initialize a respository? Why do you need to do it?

When you initialize a respository, it will keep track of all commits and allow for version control. This is why it's needed.

How is the staging area different from the working directory and the repository? What value do you think it offers?

It allows for changes, to commit certain files at a time instead of taking an entire snapshot.

How can you use the staging area to make sure you have one commit per logical change?

You can add and remove files from the staging area so that only one file is committed.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

Branches can allow for different variations of the files whilst having an organised log of each committed variation.

How do the diagrams help you visualize the branch structure?
